# Pitwall Load Simulator

Generates realistic concurrent user sessions against the deployed Cloud Run app to produce observability data in Datadog (APM traces, LLMObs spans, structured logs).

**Stack:** [Locust](https://locust.io) for load orchestration · Gemini `gemini-2.5-flash-lite` for question generation

---

## Quick Start

### 1. Install dependencies

```bash
cd load-test
python3 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
```

### 2. Configure environment

Create a `.env` file in the `load-test/` directory:

```env
GEMINI_API_KEY=your_gemini_api_key_here
GEMINI_MODEL=gemini-2.5-flash-lite
USE_GEMINI_QUESTIONS=false   # set true to generate questions live (slower)
```

> The `.env.local` file in the project root is also auto-loaded if present.

### 3. (Optional) Pre-generate question bank

Pre-generate a diverse bank of 120 F1 questions using Gemini before running the sim.
This avoids per-request Gemini latency during load generation:

```bash
python generate_questions.py --count 120
```

This writes `questions_bank.json` which `locustfile.py` loads automatically.

## Running Simulations

### Web UI (recommended for demos)

```bash
locust -f locustfile.py --host https://box-box-bits-ai-449012790678.asia-southeast1.run.app
```

Open **http://localhost:8089**, then configure:

| Setting | Recommended for Demo |
|---|---|
| Number of users | 10–20 |
| Spawn rate (users/sec) | 2 |
| Host | (pre-filled from CLI) |

---

### Headless CLI presets

**Light warm-up** (5 users, 2 min — just to seed some traces)
```bash
locust -f locustfile.py \
  --host https://box-box-bits-ai-449012790678.asia-southeast1.run.app \
  -u 5 -r 1 --run-time 2m --headless
```

**Demo traffic** (15 concurrent users, 10 min)
```bash
locust -f locustfile.py \
  --host https://box-box-bits-ai-449012790678.asia-southeast1.run.app \
  -u 15 -r 2 --run-time 10m --headless
```

**Stress test** (50 users, spike to explore rate limits)
```bash
locust -f locustfile.py \
  --host https://box-box-bits-ai-449012790678.asia-southeast1.run.app \
  -u 50 -r 5 --run-time 5m --headless
```

**Live Gemini question generation** (slower, richer variety)
```bash
USE_GEMINI_QUESTIONS=true \
locust -f locustfile.py \
  --host https://box-box-bits-ai-449012790678.asia-southeast1.run.app \
  -u 10 -r 1 --run-time 10m --headless
```

## User Types

| Class | Endpoint | Weight | Wait time |
|---|---|---|---|
| `PitwallChatUser` | `POST /api/pitwall` | 3× | 2–8 s between messages |
| `DreamTeamUser` | `POST /api/evaluate-team` | 1× | 15–45 s between submissions |

Both users get unique `userId`, `username` (generated by Faker), and `sessionId` per virtual user — so Datadog will show distinct user traces.

### Task distribution for `PitwallChatUser`

| Task | Weight | Description |
|---|---|---|
| `chat_static_question` | 5 | Random question from static bank + `questions_bank.json` |
| `chat_gemini_question` | 2 | Gemini-generated question (or static if `USE_GEMINI_QUESTIONS=false`) |
| `chat_followup` | 1 | Short follow-up to simulate a real conversation thread |

## Generate Questions Only

```bash
# Default: 100 questions across all F1 topics
python generate_questions.py

# More questions, specific topics
python generate_questions.py --count 200 --topics "race strategy,driver comparisons,technical regulations"

# Custom model and concurrency
python generate_questions.py --count 150 --model gemini-2.5-flash --concurrency 8

# Custom output file
python generate_questions.py --output my_questions.json
```

## Locust Web UI Metrics

The Locust dashboard at `http://localhost:8089` shows:

- **RPS** (requests per second) — control with `-r` spawn rate and user count
- **Response time** — p50 / p95 / p99 latency per endpoint
- **Failure rate** — 429s from Cloud Run will surface here
- **Charts** — live RPS and response time over the run

## Datadog Observability

After a simulation run, check:

| Datadog Product | What You'll See |
|---|---|
| **APM** | `api.pitwall.chat` and `api.dream_team.evaluate` spans with user tags |
| **LLM Observability** | `pitwall_chat` and `dream_team_game_evaluation` spans, token counts, latency |
| **Logs** | `event_type: pitwall_chat` and `event_type: dream_team_game_evaluation` entries |
| **Service Map** | Cloud Run → Gemini API dependency visible |

Useful Datadog log queries:
```
event_type:pitwall_chat
event_type:dream_team_game_evaluation
@scoring.synergy_class:TOXIC
@scoring.final_score:>600
```
